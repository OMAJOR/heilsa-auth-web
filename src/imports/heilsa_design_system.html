Adding a new card — checklist
1. Check first — does an existing component cover it?MetricCard, InsightCard, WearableCard, LimitCard, or MealScanCard should be reused before creating new ones.
2. Token only — no hardcoded colors or spacingAll values must come from the token file. No random hex values, no px magic numbers.
3. States are requiredEvery data-dependent card must have: loading (shimmer), empty state (icon + message), and error state (error color + retry).
4. Icon rulesTabler outline only. 20px inside cards, 22px in navigation. One icon family throughout.
5. Mobile firstTest at 375px. No overflow. Safe area insets on bottom nav. No fixed elements except bottom nav.
6. Premium gatingGold accent only. Use the PremiumLimitCard pattern. Never use gold for non-premium elements.
7. No clutter ruleMax 2 actions per card. No more than 3 metrics in a single card row. Section headers always have a title and optional "See all" — nothing else.
8. Brand gradient ruleThe teal→blue→violet gradient is reserved for the logo background, loading bars, and premium upgrade CTAs only. Do not use it decoratively.
What NOT to do
Mix Tabler + Heroicons + SF Symbols in the same screen
Hardcode #3ECFB2 directly in a component — use the token
Use the gold / premium color for non-premium elements
Ship a card without loading / empty states
Add a one-off card style that can't be reused
Use the brand gradient as a card background
